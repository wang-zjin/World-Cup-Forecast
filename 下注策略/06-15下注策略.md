## 最大收益风险比组合

考虑下注资金 B 应该如何分配以最大化收益风险比：

```text
组合期望收益 / 组合收益标准差
```

设一共有 n 个候选标的，第 i 个标的的下注资金占比为 w_i，则：

```text
w = (w_1, w_2, ..., w_n)

sum(w_i) = 1
w_i >= 0
```

每个标的的单注净收益 R_i 定义为：

```text
若事件命中：R_i = O_i - 1
若事件未命中：R_i = -1
```

其中 O_i 是体彩十进制赔率，p_i 是 Polymarket 给出的概率估计。单个标的的期望净收益为：

```text
mu_i = E[R_i] = p_i * O_i - 1
```

把所有标的的期望收益写成向量 mu，把收益协方差矩阵写成 Sigma，则组合的期望收益和方差为：

```text
组合期望收益 = w^T * mu
组合收益方差 = w^T * Sigma * w
组合收益标准差 = sqrt(w^T * Sigma * w)
```

因此，下注资金占比 w 由下面的优化问题得到：

```text
maximize_w    (w^T * mu) / sqrt(w^T * Sigma * w)

subject to    sum(w_i) = 1
              w_i >= 0
```

协方差矩阵 Sigma 的处理方式：

```text
同一场比赛的不同比分互斥，不能同时命中；
不同比赛之间近似按独立事件处理。
```

为了避免混淆，先定义指示变量 X_i：

```text
如果事件 i 命中，X_i = 1
如果事件 i 未命中，X_i = 0
```

则第 i 个标的的净收益可以写成：

```text
R_i = O_i * X_i - 1
```

因此，对角线元素是单个标的自身的方差：

```text
Sigma_ii = Var(R_i) = p_i * (1 - p_i) * O_i^2
```

非对角线元素可以从指示变量直接推导：

```text
Sigma_ij = Cov(R_i, R_j)
         = O_i * O_j * Cov(X_i, X_j)
         = O_i * O_j * (E[X_i * X_j] - p_i * p_j)
```

当 i = j 时，对角线元素是单个标的自身的方差：

```text
E[X_i * X_i] = E[X_i] = p_i
Sigma_ii = O_i^2 * (p_i - p_i^2)
         = p_i * (1 - p_i) * O_i^2
```

当比赛不同时，近似看成独立事件：

```text
E[X_i * X_j] = p_i * p_j
Sigma_ij = 0
```

当属于同一场比赛的两个不同比分时，X_i 和 X_j 不能同时发生：

```text
E[X_i * X_j] = 0
Sigma_ij = -O_i * O_j * p_i * p_j
```

例如德国 vs 库拉索的 0:1 和 1:1 的协方差约为 `-100.00 * 32.00 * 0.0170 * 0.0390 = -2.1216`；科特迪瓦 vs 厄瓜多尔的 2:0 和 1:0 的协方差约为 `-15.00 * 7.25 * 0.0700 * 0.1400 = -1.06575`。

优化求解代码如下：

```python
import numpy as np
from scipy.optimize import minimize

mu = np.array([0.475, 0.176, 0.248, 0.700, 0.050, 0.015])

Sigma = np.array([
    [34.699375, 0.000000,  0.000000,  0.000000,   0.000000,  0.000000],
    [ 0.000000,23.313024,  0.000000,  0.000000,   0.000000,  0.000000],
    [ 0.000000, 0.000000, 38.378496, -2.121600,   0.000000,  0.000000],
    [ 0.000000, 0.000000, -2.121600,167.110000,   0.000000,  0.000000],
    [ 0.000000, 0.000000,  0.000000,  0.000000,  14.647500, -1.065750],
    [ 0.000000, 0.000000,  0.000000,  0.000000,  -1.065750,  6.328525],
])

def neg_reward_risk_ratio(w):
    expected_return = w @ mu
    variance = w @ Sigma @ w
    if variance <= 0:
        return 1e9
    return -expected_return / np.sqrt(variance)

constraints = [
    {"type": "eq", "fun": lambda w: np.sum(w) - 1},
]
bounds = [(0, 1)] * len(mu)

initial_points = [np.ones(len(mu)) / len(mu)]
for i in range(len(mu)):
    point = np.zeros(len(mu))
    point[i] = 1
    initial_points.append(point)

rng = np.random.default_rng(42)
for _ in range(100):
    initial_points.append(rng.dirichlet(np.ones(len(mu))))

best = None
for x0 in initial_points:
    result = minimize(
        neg_reward_risk_ratio,
        x0=x0,
        method="SLSQP",
        bounds=bounds,
        constraints=constraints,
        options={"maxiter": 1000, "ftol": 1e-12},
    )
    if result.success and (best is None or result.fun < best.fun):
        best = result

w = best.x
expected_return = w @ mu
standard_deviation = np.sqrt(w @ Sigma @ w)
reward_risk_ratio = expected_return / standard_deviation

print("w =", np.round(w, 6))
print("expected_return =", round(expected_return, 6))
print("standard_deviation =", round(standard_deviation, 6))
print("reward_risk_ratio =", round(reward_risk_ratio, 6))
```

优化问题具体为：

```text
maximize_w
  (0.475*w_1 + 0.176*w_2 + 0.248*w_3 + 0.700*w_4 + 0.050*w_5 + 0.015*w_6)
  /
  sqrt(w^T * Sigma * w)

subject to
  w_1 + w_2 + w_3 + w_4 + w_5 + w_6 = 1
  w_1, w_2, w_3, w_4, w_5, w_6 >= 0
```

这个优化问题没有要求某个标的一定下注，只是在所有非负权重中寻找“单位波动对应期望收益最高”的组合。由于目标函数对整体缩放不敏感，所以先用 `sum(w_i) = 1` 得到下注资金 B 内部的相对分配，再用 Kelly 决定 B 应该占总本金多少。

## 全事件纳入后的组合优化

上面的组合只使用了超额期望收益为正的 6 个事件。进一步可以把完整可比项目表里的全部 29 个事件都纳入优化，包括超额期望收益为负的事件。负期望事件本身不值得单独买入，但如果它和组合中已有事件存在负相关，理论上可能降低组合方差，从而改善收益风险比。

优化目标沿用上一节：

```text
maximize_w    (w^T * mu) / sqrt(w^T * Sigma * w)

subject to    sum(w_i) = 1
              w_i >= 0
```

其中：

```text
mu_i = p_i * O_i - 1
Sigma_ij = O_i * O_j * (E[X_i * X_j] - p_i * p_j)
```

协方差处理规则：

```text
不同比赛：近似独立，E[X_i * X_j] = p_i * p_j
同场同玩法不同结果：互斥，E[X_i * X_j] = 0
同场比分与胜负：如果比分属于该胜负结果，则 E[X_i * X_j] = p_比分；否则为 0
```

优化求解代码如下。`events` 使用上方完整可比项目表中的 29 个事件，每个元素依次为：比赛、玩法、事件、体彩赔率、Polymarket 概率。

```python
import numpy as np
from scipy.optimize import minimize

events = [
    ("德国 vs 库拉索", "比分", "0:1", 100.00, 0.0170),
    ("德国 vs 库拉索", "比分", "1:1", 32.00, 0.0390),
    ("德国 vs 库拉索", "比分", "2:1", 19.00, 0.0400),
    ("德国 vs 库拉索", "比分", "3:0", 5.05, 0.1700),
    ("德国 vs 库拉索", "比分", "3:1", 12.00, 0.0610),
    ("荷兰 vs 日本", "胜负", "胜", 1.88, 0.4800),

def score_result(score):
    home, away = map(int, score.split(":"))
    if home > away:
        return "胜"
    if home == away:
        return "平"
    return "负"

def joint_probability(i, j):
    match_i, type_i, event_i, _, p_i = events[i]
    match_j, type_j, event_j, _, p_j = events[j]
    if i == j:
        return p_i
    if match_i != match_j:
        return p_i * p_j
    if type_i == "比分" and type_j == "比分":
        return 0.0
    if type_i == "胜负" and type_j == "胜负":
        return 0.0
    if type_i == "比分" and type_j == "胜负":
        return p_i if score_result(event_i) == event_j else 0.0
    if type_i == "胜负" and type_j == "比分":
        return p_j if event_i == score_result(event_j) else 0.0
    return p_i * p_j

odds = np.array([event[3] for event in events])
prob = np.array([event[4] for event in events])
mu = prob * odds - 1

n = len(events)
Sigma = np.zeros((n, n))
for i in range(n):
    for j in range(n):
        e_xixj = joint_probability(i, j)
        Sigma[i, j] = odds[i] * odds[j] * (e_xixj - prob[i] * prob[j])

def neg_reward_risk_ratio(w):
    expected_return = w @ mu
    variance = w @ Sigma @ w
    if variance <= 0:
        return 1e9
    return -expected_return / np.sqrt(variance)

constraints = [{"type": "eq", "fun": lambda w: np.sum(w) - 1}]
bounds = [(0, 1)] * n

rng = np.random.default_rng(42)
initial_points = [np.ones(n) / n]
for i in range(n):
    point = np.zeros(n)
    point[i] = 1
    initial_points.append(point)
for _ in range(1000):
    initial_points.append(rng.dirichlet(np.ones(n)))

best = None
for x0 in initial_points:
    result = minimize(
        neg_reward_risk_ratio,
        x0=x0,
        method="SLSQP",
        bounds=bounds,
        constraints=constraints,
        options={"maxiter": 5000, "ftol": 1e-13},
    )
    if result.success and (best is None or result.fun < best.fun):
        best = result

w = best.x
print("期望收益率 =", round(w @ mu, 6))
print("收益标准差 =", round(np.sqrt(w @ Sigma @ w), 6))
print("收益风险比 =", round((w @ mu) / np.sqrt(w @ Sigma @ w), 6))
for weight, event in sorted(zip(w, events), reverse=True):
    if weight > 0.0001:
        print(round(weight, 6), event)
```

按体彩比分盘口，每场比赛有 31 个比分桶：

```text
12 个主胜具体比分 + 胜其他
4 个平局具体比分 + 平其他
12 个客胜具体比分 + 负其他
合计 31 个比分桶
```

如果要用 Polymarket 的交易量分布反推比分真实分布，设每一场比赛的真实概率为：

```text
x_s = 体彩比分桶 s 的真实概率
sum_s x_s = 1
x_s >= 0
```

Polymarket 中每个被 `纳入` 的盘口事件 k 都可以转成一个对比分桶集合的约束。设该事件对应的比分集合为 S_k，Polymarket 隐含概率为 q_k，该事件所在 market 的 24 小时交易量为 V_k，则：

```text
q_k = 1 / Polymarket隐含赔率
估算方向支持金额 A_k = V_k * q_k
模型预测概率 P_k = sum_{s in S_k} x_s
```

例如 `科特迪瓦 vs 厄瓜多尔 | 全场大小球（1.5） | 大` 表示“总进球数至少 2”。其投资支持金额可近似写为：

```text
4885.215462 * (1 / 1.67)
```

一个可执行的估计框架是加权最小二乘：

```text
min_x    sum_k A_k * (sum_{s in S_k} x_s - q_k)^2

subject to
         sum_s x_s = 1
         x_s >= 0
```

其中 `A_k` 使用估算方向支持金额而不是直接使用 market 总交易量，是为了避免把同一个二元 market 的两边都赋予完整交易量。例如大小球同一盘口的“大 / 小”共享同一个 `volume24h`，建模时用 `V_k * q_k` 把 market 交易量近似拆到对应方向。建模时只使用表中 `处理=纳入` 的事件，并按玩法把它们转换成比分桶集合 S_k，例如胜负、具体比分、总进球区间、双方是否进球、净胜球条件。

## 比分真实概率的优化求解

为了让胜负、比分、全场大小球、双方进球、让球盘都能映射到同一套变量，这里不再直接用体彩的 31 个聚合比分桶求解，而是用具体比分网格：

```text
x[h, a] = 主队进 h 球、客队进 a 球的真实概率
h, a = 0, 1, ..., 10
sum_{h,a} x[h, a] = 1
x[h, a] >= 0
```

这样做的好处是：让球盘可以直接写成净胜球条件，大小球可以直接写成总进球条件，最后结果也能按“主队进球数 × 客队进球数”的矩阵展示。这个版本只建模单队进球数 0 到 10，单队进球数超过 10 的尾部做截断近似；足球比分尾部通常很小，但德国 vs 库拉索这种大比分场景仍要把这个假设记住。

完整可复现脚本见 `scripts/estimate_score_distribution.py`。核心优化代码如下：

```python
GMAX = 10
RIDGE = 0.02
GOALS = tuple(range(GMAX + 1))
SCORES = tuple((h, a) for h in GOALS for a in GOALS)

# 每个被纳入的 Polymarket 事件 k 转成一个 0/1 mask：
# mask_k[h, a] = 1 表示比分 h:a 会使事件 k 发生，否则为 0。
M = np.vstack([event["mask"] for event in events])
q = np.array([event["q"] for event in events])
A = np.array([event["weight"] for event in events])
w = A / A[A > 0].mean()

# 先拟合一个 Poisson 先验，只用于在欠定问题中选择更平滑的解。
# 真实约束仍然来自 M、q、A。
home_lambda, away_lambda = fit_poisson_prior(M, q, w)
prior = poisson_prior(home_lambda, away_lambda)

def objective(x):
    market_error = np.sum(w * (M @ x - q) ** 2)
    smooth_penalty = np.sum((x - prior) ** 2 / (prior + 1e-9))
    return market_error + RIDGE * smooth_penalty

result = minimize(
    objective,
    x0=prior,
    method="SLSQP",
    bounds=[(0.0, 1.0)] * len(SCORES),
    constraints={"type": "eq", "fun": lambda x: np.sum(x) - 1.0},
    options={"ftol": 1e-12, "maxiter": 2000},
)

x = result.x.reshape((GMAX + 1, GMAX + 1))
```

其中 `q_k` 使用 Polymarket 原始价格，`A_k = V_k * q_k` 使用“market 24h交易量 × 该方向价格”近似该方向上的投资支持金额。由于 121 个具体比分变量多于可用盘口约束，问题本身是欠定的，所以加入很小的 Poisson 先验正则项；这个正则项只是在多个近似满足盘口的解中选择一个更平滑的解。

## 最优投注方案

参考前面的组合优化框架，这里允许组合里保留现金仓位，即可以选择不下注。设第 i 个体彩事件的单位净收益为 R_i，下注权重为 f_i：

```text
f_i >= 0
sum_i f_i <= 1
现金权重 = 1 - sum_i f_i
```

如果用 Kelly 最优增长写法，目标是最大化：

```text
maximize_f    E[ log(1 + sum_i f_i * R_i) ]
```

在 f = 0 附近，沿任一事件 i 增加极小仓位的边际收益是：

```text
d/dε E[log(1 + ε * R_i)] | ε=0 = E[R_i] = Polymarket估计概率 * 体彩赔率 - 1
```

上一节计算得到的 145 个可投体彩数字盘口中，正期望盘口数为 0。因此所有事件在 f = 0 的边际收益都是负数，最优解就是不下注。用前面的收益风险比框架也会得到同样的投资含义：没有正期望资产时，现金 100% 支配任何长仓投注组合。

完整可复现脚本见 `scripts/build_optimal_betting_plan.py`，输出如下：

候选体彩数字盘口数：145
正期望盘口数：0

最优投注方案：不下注，现金仓位 100%。

| 仓位 | 权重 | 说明 |
|---|---:|---|
| 现金 / 不下注 | 100.0% | 所有可投体彩盘口的估计期望收益均小于 0 |
| 体彩事件组合 | 0.0% | 无正期望事件，不构建风险仓位 |

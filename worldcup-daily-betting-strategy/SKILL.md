# 世界杯每日下注策略

## 核心口径

以本项目本地文件为准生成策略，不把联网查询作为默认步骤：

```text
2026世界杯体彩赔率.md
2026世界杯polymarket赔率.md
2026世界杯赛程_北京时间.md
scripts/estimate_score_distribution.py
scripts/compare_sporttery_expected_returns.py
scripts/build_optimal_betting_plan.py
06-15下注策略.md
```

把体彩赔率视为可成交十进制赔率，把 Polymarket 价格视为市场概率信息。每日下注默认面向“北京时间下一天”的比赛；先判断当前北京时间，再确定目标日期。报告必须同时输出两套策略结果：

- 策略一：简化版分布，只使用体彩和 Polymarket 能直接一一对应的同场、同玩法、同事件数据。
- 策略二：考虑增量信息的分布，使用可由全场比分判断的 Polymarket 事件反推比分分布，再计算体彩所有数字盘口的期望收益。

正式执行建议以策略二为主；策略一保留为对照和解释。若两套策略结论冲突，报告中明确写出冲突并采用更保守的执行口径。

## 每日流程

1. 判断当前北京时间，并写入报告。默认目标日期是当前北京时间日期的下一天，例如北京时间 `2026-06-15` 生成 `2026-06-16` 的下注策略。只有用户明确要求复盘或指定日期时，才覆盖默认目标日期。
2. 按目标日期命名输出文件：`MM-DD下注策略.md`，例如目标日期 `2026-06-16` 对应 `06-16下注策略.md`。不要用生成当天日期命名次日策略。
3. 检查数据文件是否覆盖目标日期。若用户要求“最新/刷新”，先使用项目内 `$sporttery-football-odds-fetch` 和 `$polymarket-worldcup-odds-fetch`，或运行它们指向的脚本刷新数据。联网刷新失败时按 Codex 权限规则请求批准，不要绕过审批。
4. 若脚本仍写死旧日期或旧目标场次，先同步切换到目标日期后再生成策略；报告里写清楚脚本当前覆盖情况。
5. 先输出策略一：简化版分布。使用同口径可比项做期望收益表和组合优化，保留正期望候选、连续权重、离散注数、不下注结论。
6. 再输出策略二：考虑增量信息的分布。运行比分分布估计：

```bash
python3 scripts/estimate_score_distribution.py
```

记录每场的纳入事件数、Poisson 先验 lambda、目标函数值；只在报告中保留必要摘要，完整矩阵按用户需要展开。

7. 运行体彩期望收益对比：

```bash
python3 scripts/compare_sporttery_expected_returns.py
```

按 `期望收益 = Polymarket估计概率 * 体彩赔率 - 1` 排序。只把有数字赔率的体彩盘口纳入，跳过 `-`。

8. 生成投注方案：

```bash
python3 scripts/build_optimal_betting_plan.py
```

如果脚本显示正期望盘口数为 0，正式方案必须是 `不下注，现金仓位 100%`。若用户要求强制试投，单独列为“验证模型的小额成本”，不能写成推荐。

9. 若存在正期望盘口，按下面的组合优化规则计算投注权重；不要只按单项期望收益排序下注。

## 时间判断

制定策略前先取得北京时间：

```bash
TZ=Asia/Shanghai date '+%Y-%m-%d %H:%M:%S %Z'
```

目标日期规则：

```text
默认目标日期 = 当前北京时间日期 + 1 天
输出文件名 = 目标日期的 MM-DD下注策略.md
```

如果目标日期比赛已经开赛或完赛，策略不可作为可执行投注；报告仍可复盘，但必须写明“不可执行”。如果目标日期没有比赛，改为下一批有比赛的北京时间日期，并说明跳过原因。

## 策略一：简化版分布

复刻 `06-15下注策略.md` 前半部分的简化思路，先做同口径可比项：

- 同一场比赛
- 同一玩法
- 同一事件
- 两边都有数字赔率
- 体彩让球盘和 Polymarket 让球盘盘口线必须一致，否则排除

直接超额收益：

```text
超额期望收益 = 体彩赔率 / Polymarket隐含赔率 - 1
```

单项期望收益：

```text
期望收益 = Polymarket 概率 * 体彩赔率 - 1
```

输出要求：

- 完整可比项目表，至少包含比赛、玩法、事件、体彩赔率、Polymarket 隐含赔率、期望收益、处理。
- 候选标的表，只纳入正期望事件。
- 最大收益风险比组合；若没有正期望事件，输出现金 100%。
- 离散注数方案；若是负期望强制试投，标注为非推荐。
- 一段以 `完整投注方案如下：` 开头的自然语言执行清单。

这一步只能说明体彩相对 Polymarket 同事件赔率是否更高。若策略二给出不同概率，以策略二作为正式执行依据。

## 策略二：考虑增量信息的分布

复刻 `06-15下注策略.md` 后半部分的增量信息思路：不仅使用胜负和比分，还纳入所有能由最终全场比分判断是否发生的 Polymarket 事件，包括胜负、比分、全场大小球、双方进球、可转成净胜球条件的让球盘。排除半场、下半场、先进球球队、角球、球员数据等不能只由全场比分唯一反映的事件。

输出要求：

- Polymarket 增量事件筛选口径和数量摘要。
- 每场全场比分分布估计摘要：纳入事件数、Poisson 先验 lambda、目标函数值。
- 体彩所有数字盘口期望收益表，至少展示前 12 个；如果篇幅允许，保留完整表。
- 最优投注方案：现金仓位、事件组合权重、离散注数、总投入、期望盈亏。
- 一段以 `完整投注方案如下：` 开头的自然语言执行清单。
- 与策略一的结论对比：一致、冲突、或策略二推翻策略一。

## 组合优化

单个事件净收益：

```text
若命中：R_i = O_i - 1
若未中：R_i = -1
mu_i = p_i * O_i - 1
```

其中 `O_i` 是体彩赔率，`p_i` 是 Polymarket 估计概率。正期望事件为 `mu_i > 0`。

连续权重优化：

```text
maximize_w    (w^T * mu) / sqrt(w^T * Sigma * w)
subject to    sum(w_i) = 1
              w_i >= 0
```

允许现金仓位时：

```text
f_i >= 0
sum_i f_i <= 1
现金权重 = 1 - sum_i f_i
```

若所有 `mu_i <= 0`，现金 100% 支配任何长仓投注组合。

协方差规则：

- 同一事件：`Sigma_ii = p_i * (1 - p_i) * O_i^2`
- 不同比赛：近似独立，`Sigma_ij = 0`
- 同场同玩法不同结果：互斥，`Sigma_ij = -O_i * O_j * p_i * p_j`
- 同场比分与胜负：如果比分属于该胜负结果，则 `E[X_i * X_j] = p_比分`；否则为 0

## 注数离散化

体彩默认每注 2 元。若用户给定预算，按连续权重换算注数并做整数取整，至少输出：

```text
注数
投注金额
体彩赔率
Polymarket估计概率
期望收益
期望盈亏
```

若用户未给预算且存在正期望组合，使用最小成本法：以保留事件中最小连续权重为 1 注，其他事件按相对权重四舍五入，最低 1 注。报告总投入、期望盈利、最高可能返奖，并说明取整会偏离连续最优权重。

## 完整投注方案文本块

每一个策略都必须在表格之后输出一段自然语言执行清单。标题和格式固定：

```text
完整投注方案如下：

德国 vs 库拉索，比分 1:1，买 2 注。

德国 vs 库拉索，比分 0:1，买 1 注。

荷兰 vs 日本，比分 0:2，买 3 注。

科特迪瓦 vs 厄瓜多尔，比分 2:0，买 1 注。

科特迪瓦 vs 厄瓜多尔，比分 1:0，买 1 注。

瑞典 vs 突尼斯，比分 0:2，买 5 注。

每注 2 元，合计 13 注，总投入 26 元。
```

生成规则：

- 每个下注事件独占一段，格式为：`{比赛}，{玩法} {事件}，买 {注数} 注。`
- 若玩法是 `比分`，写成 `比分 0:2`；若玩法是 `胜负`，写成 `胜负 胜`；若玩法是 `让负（-1）`，写成 `让负（-1） 胜`。
- 这段必须按比赛排序：优先按 `2026世界杯赛程_北京时间.md` 中目标日期的北京时间开球时间从早到晚排序；如果缺少赛程时间，就按赔率表中目标日期首次出现的比赛顺序排序。
- 同一场比赛内，把该场所有下注项放在一起；同场多项按玩法顺序 `胜负`、`让负`、`比分` 排序，同玩法内按注数从多到少排序，注数相同则按事件在策略表中的顺序。
- 最后一段写清每注金额、总注数和总投入。
- 策略一和策略二都要各自输出这段，即使两套策略的投注方案相同也不要省略。
- 如果某套策略的正式结论是不下注，也仍要输出：

```text
完整投注方案如下：

不下注，现金仓位 100%。

每注 2 元，合计 0 注，总投入 0 元。
```

- 如果用户要求强制试投但策略正式结论是不下注，先输出正式不下注版本，再单独以“若强制试投”列出试投注数，不要把试投写成推荐。

## 报告结构

输出 Markdown，文件名默认沿用项目习惯：

```text
MM-DD下注策略.md
```

若同名文件已存在，先确认用户是否要覆盖；更新同一目标日期已有策略时可以直接编辑该文件。

报告建议结构：

```text
# MM-DD下注策略

> 说明：本策略只基于当前本地赔率文件与项目脚本的静态数据。Polymarket 价格被视为概率估计，体彩赔率被视为可成交赔率；任一假设失真都会影响期望收益和下注比例。本报告生成前已判断北京时间，默认面向下一天比赛。

## 原始数据与口径
## 策略一：简化版分布
### 原始可比项目
### 候选标的
### 最大收益风险比组合
### 策略一投注方案
### 策略一完整投注方案
## 策略二：考虑增量信息的分布

```

正式建议要放在“两套策略结论对比”和“风险提示与执行口径”中，且明确写出：

- 是否下注
- 现金仓位
- 每个事件的注数和金额
- 总投入
- 期望盈亏
- 如果是负期望强制试投，标注为非推荐
- 目标日期是否仍可执行投注
- 每套策略各自的 `完整投注方案如下：` 文本块

## 风控要求

- 不要把负期望事件写成推荐。
- 不要把 `Polymarket 隐含赔率低于体彩赔率` 直接等同于正式正期望；必须同时输出策略一和策略二，并以策略二作为主执行依据。
- 不要比较盘口线不同的让球盘。
- 不要隐藏数据缺口；缺少 Polymarket、体彩或赛程匹配时，在报告中列出未覆盖比赛。
- 不要输出没有当前北京时间、目标日期、数据文件版本或可复现命令的策略。
- 不要只输出单一策略；每日策略必须同时包含简化版分布结果和考虑增量信息的分布结果。
- 不要只输出表格而缺少自然语言完整投注方案；策略一和策略二都必须有 `完整投注方案如下：` 段落。

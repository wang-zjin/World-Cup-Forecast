# 体彩足球赔率数据获取

## 目标

从 `https://m.sporttery.cn/mjc/jsq/zqspf/` 获取竞彩足球赔率，更新项目里的体彩赔率表。优先使用 `体彩赔率.md`；如果不存在，就查找当前目录里的 `*体彩赔率.md`，例如 `2025世界杯体彩赔率.md`。

本项目约定：

- 本项目已有脚本 `scripts/update_sporttery_odds.py`；刷新赔率时优先运行脚本，不要重复手写抓取和写回逻辑
- `胜负` 对应体彩接口的 `HAD`
- `让负（让球数）` 对应体彩接口的 `HHAD`
- `比分` 对应体彩接口的 `CRS`
- `进球数` 对应体彩接口的 `TTG`
- 没有开售、接口缺失、历史比赛获取不到、单个事件无赔率时，赔率统一写成 `-`
- 不要用 `0.00` 表示缺失赔率

## 页面到接口的发现方法

移动端页面本身只是 HTML 壳，赔率由前端脚本调用接口渲染。

1. 打开页面源码：

```bash
curl -L 'https://m.sporttery.cn/mjc/jsq/zqspf/'
```

2. 在源码里可以看到接口根域和脚本：

```js
jsCommonDataV1.webApi = "//webapi.sporttery.cn"
```

关键脚本：

```text
https://static.sporttery.cn/res_1_0/jcwm/default/jc/jsq/dataTransfer.js
https://static.sporttery.cn/res_1_0/jcwm/default/jc/jsq/lotJs.js
https://static.sporttery.cn/res_1_0/common/js/commonV1.js
```

3. `dataTransfer.js` 里的 `getJsqMatchDate(pool, stype)` 调用足球计算器接口：

```text
https://webapi.sporttery.cn/gateway/uniform/football/getMatchCalculatorV1.qry?channel=c&poolCode=had,hhad,crs,ttg
```

`commonV1.js` 里 `comDataChannel` 是 `c`，表示计算器数据渠道。

## 正式请求

直接裸请求接口容易被腾讯云 WAF 拦截。按页面 AJAX 行为带移动端 UA、Referer、Origin 和 X-Requested-With。

```bash
curl -L 'https://webapi.sporttery.cn/gateway/uniform/football/getMatchCalculatorV1.qry?channel=c&poolCode=had,hhad,crs,ttg' \
  -H 'User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 17_5 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.5 Mobile/15E148 Safari/604.1' \
  -H 'Accept: application/json, text/javascript, */*; q=0.01' \
  -H 'Referer: https://m.sporttery.cn/mjc/jsq/zqspf/' \
  -H 'Origin: https://m.sporttery.cn' \
  -H 'X-Requested-With: XMLHttpRequest' \
  -o /tmp/sporttery_calc.json
```

如果返回 HTML 且标题包含 `WAF拦截页面`，说明请求头不完整或访问方式被拦截，优先补齐以上请求头后重试。

## 返回结构

接口返回 JSON，主要字段：

```text
value.lastUpdateTime
value.matchInfoList[].businessDate
value.matchInfoList[].subMatchList[]
```

单场比赛字段：

```text
matchDate          比赛日期；按体彩页面展示日期匹配本项目北京时间日期
matchTime          开球时间
matchNumStr        如 周日011
leagueAbbName      赛事名
homeTeamAbbName    主队简称
awayTeamAbbName    客队简称
poolList           开售玩法列表
had                胜平负赔率对象，可能是 {}
hhad               让球胜平负赔率对象，含 goalLine
crs                比分赔率对象
ttg                总进球数赔率对象
```

匹配本地表时优先使用：

```text
{matchDate} {matchTime前5位} + "{homeTeamAbbName} vs {awayTeamAbbName}"
```

如果本地赛程与体彩队名存在译名差异，先人工建立队名映射，再更新。

## 玩法映射

`had` 和 `hhad` 的胜平负字段一致：

```text
胜 -> h
平 -> d
负 -> a
```

`hhad.goalLine` 是让球数。写回赔率表时，将类型改成：

```text
让负（{goalLine}）
```

例如：

```text
让负（-3）
让负（+1）
```

如果 `had` 是 `{}`，说明该场胜负玩法没有可用赔率，三行都写 `-`。

## 比分映射

`crs` 字段到本地 `事件` 的映射：

```text
1:0 -> s01s00
2:0 -> s02s00
2:1 -> s02s01
3:0 -> s03s00
3:1 -> s03s01
3:2 -> s03s02
4:0 -> s04s00
4:1 -> s04s01
4:2 -> s04s02
5:0 -> s05s00
5:1 -> s05s01
5:2 -> s05s02
胜其他 -> s1sh

0:0 -> s00s00
1:1 -> s01s01
2:2 -> s02s02
3:3 -> s03s03
平其他 -> s1sd

0:1 -> s00s01
0:2 -> s00s02
1:2 -> s01s02
0:3 -> s00s03
1:3 -> s01s03
2:3 -> s02s03
0:4 -> s00s04
1:4 -> s01s04
2:4 -> s02s04
0:5 -> s00s05
1:5 -> s01s05
2:5 -> s02s05
负其他 -> s1sa
```

若某个比分字段不存在、为空字符串、对应玩法对象为空，写 `-`。

## 进球数映射

`ttg` 字段到本地 `事件` 的映射：

```text
0球 -> s0
1球 -> s1
2球 -> s2
3球 -> s3
4球 -> s4
5球 -> s5
6球 -> s6
7+球 -> s7
```

若某个进球数字段不存在、为空字符串、对应玩法对象为空，写 `-`。

## 更新规则

1. 只更新用户指定日期或指定比赛，不要重写整张表的结构。
2. 已获取到的赔率按接口字符串原样写入，例如 `1.73`、`1000`、`900.0`。
3. 获取不到的赔率写 `-`。
4. `HHAD` 要同时更新赔率和类型里的让球数。
5. `TTG` 写入 `进球数` 的 `0球` 到 `7+球` 八个事件。
6. 写入后检查：

```bash
rg -n '\| 0\.00 \|' <赔率表文件.md>
```

该命令不应返回结果。

## 参考更新脚本片段

以下片段假设接口 JSON 已保存到 `/tmp/sporttery_calc.json`，并且当前目录有赔率表 Markdown 文件。

```python
from pathlib import Path
import json

md_path = Path("体彩赔率.md")
if not md_path.exists():
    matches = sorted(Path(".").glob("*体彩赔率.md"))
    if not matches:
        raise FileNotFoundError("未找到 体彩赔率.md 或 *体彩赔率.md")
    md_path = matches[0]
data = json.loads(Path("/tmp/sporttery_calc.json").read_text(encoding="utf-8"))

result_key = {"胜": "h", "平": "d", "负": "a"}
score_key = {
    "1:0": "s01s00", "2:0": "s02s00", "2:1": "s02s01",
    "3:0": "s03s00", "3:1": "s03s01", "3:2": "s03s02",
    "4:0": "s04s00", "4:1": "s04s01", "4:2": "s04s02",
    "5:0": "s05s00", "5:1": "s05s01", "5:2": "s05s02",
    "胜其他": "s1sh",
    "0:0": "s00s00", "1:1": "s01s01", "2:2": "s02s02",
    "3:3": "s03s03", "平其他": "s1sd",
    "0:1": "s00s01", "0:2": "s00s02", "1:2": "s01s02",
    "0:3": "s00s03", "1:3": "s01s03", "2:3": "s02s03",
    "0:4": "s00s04", "1:4": "s01s04", "2:4": "s02s04",
    "0:5": "s00s05", "1:5": "s01s05", "2:5": "s02s05",
    "负其他": "s1sa",
}
total_goals_key = {
    "0球": "s0", "1球": "s1", "2球": "s2", "3球": "s3",
    "4球": "s4", "5球": "s5", "6球": "s6", "7+球": "s7",
}

api_matches = {}
for group in data["value"]["matchInfoList"]:
    for match in group.get("subMatchList", []):
        matchup = f'{match["homeTeamAbbName"]} vs {match["awayTeamAbbName"]}'
        key = (match["matchDate"], match["matchTime"][:5], matchup)
        api_matches[key] = match

new_lines = []
for line in md_path.read_text(encoding="utf-8").splitlines():
    if not line.startswith("| ") or line.startswith("| 时间 ") or line.startswith("|---"):
        new_lines.append(line)
        continue

    cells = [cell.strip() for cell in line.strip().strip("|").split("|")]
    if len(cells) != 5:
        new_lines.append(line)
        continue

    time_text, matchup, typ, event, old_odd = cells
    date = time_text.split("（", 1)[0]
    kick_time = time_text.rsplit(" ", 1)[-1]
    match = api_matches.get((date, kick_time, matchup))
    if not match:
        odd = "-"
        new_lines.append(f"| {time_text} | {matchup} | {typ} | {event} | {odd} |")
        continue

    odd = "-"
    new_typ = typ
    if typ == "胜负":
        pool = match.get("had") or {}
        odd = pool.get(result_key.get(event, ""), "-") or "-"
    elif typ.startswith("让负（"):
        pool = match.get("hhad") or {}
        if pool:
            new_typ = f'让负（{pool.get("goalLine") or "0"}）'
            odd = pool.get(result_key.get(event, ""), "-") or "-"
    elif typ == "比分":
        pool = match.get("crs") or {}
        odd = pool.get(score_key.get(event, ""), "-") or "-"
    elif typ == "进球数":
        pool = match.get("ttg") or {}
        odd = pool.get(total_goals_key.get(event, ""), "-") or "-"

    new_lines.append(f"| {time_text} | {matchup} | {new_typ} | {event} | {odd} |")

md_path.write_text("\n".join(new_lines) + "\n", encoding="utf-8")
```

按指定日期更新时，在读取本地行后先判断 `time_text.startswith("YYYY-MM-DD")`，避免把不在本次范围内的历史或未来比赛改掉。
